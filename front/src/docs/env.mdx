# Authorizer Environment Variables

Authorizer server supports the following environment variables

| Variable              | Description                                                                                                                                                                                                                                                 | Required | Default Value             |
| --------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------- | ------------------------- |
| `MYSQL_ROOT_PASSWORD` | This variable is used to specify the root password for the MySQL database server. It grants full access to all databases and actions on them.                                                                                                               | false    |                           |
| `MYSQL_DATABASE`      | This variable specifies the name of the database that will be created when the MySQL Docker container is started. In this case, it indicates the name of the database, which could be used for a WordPress installation, such as 'MyWordPressDatabaseName'. | true     | 'MyWordPressDatabaseName' |
| `MYSQL_USER`          | This variable sets the username for accessing the MySQL database. Replace with your desired username.                                                                                                                                                       | false    |                           |
| `MYSQL_PASSWORD`      | This variable is used to define the password associated with the specified MYSQL_USER Replace with the desired strong password for the MySQL user.                                                                                                          | false    |                           |

It is expected for this variable to be present as system env or `.env` at the root of project. You can also pass `env_file` as command line argument. Example:

```sh
make && ./build/server --env_file=.env.local
```
